## Four Role Types

1. **Associate** — earns commission on own billed hours
2. **Director** (maps to "manager" conceptually) — earns commission on own billed hours PLUS an override on total project revenue
3. **Analyst / Consultant / PM** — earns commission on revenue generated by associates' billed hours only (not their own)

---

## Commission Formulas (routes.py:256-273, kept in sync with models.py:208-243)

### Associate
```
direct_comm  = own_revenue * (staff.commission_percentage / 100)
override_comm = 0
```
`own_revenue` = sum of stored `HoursEntry.revenue` for this employee on this project in the date range.

### Director
```
direct_comm   = own_revenue * (staff.commission_percentage / 100)
override_comm = total_project_revenue * (employee.override_percentage / 100)
```
`total_project_revenue` = sum of ALL employees' stored revenue on the project in the date range.  
`override_percentage` lives on `Employee` model (not ProjectStaff), default 0, set to ~2% for directors.

### PM / Analyst / Consultant
```
direct_comm   = total_assoc_revenue * (staff.commission_percentage / 100)
override_comm = 0
```
`total_assoc_revenue` = sum of revenue for all employees whose role == 'associate' on this project in the date range.

---

## Key Data Flow

- `HoursEntry.revenue` is stored at creation time (not computed live), so rate changes don't rewrite history.
- `staff.commission_percentage` — per-project rate, lives on `ProjectStaff`.
- `employee.override_percentage` — global rate, lives on `Employee` (only meaningful for Directors).
- Revenue base for non-associates = **associate revenue only**, not total project revenue.
